1. **Prepare Data:** The TMDB 1 Million dataset is huge. For a Replit prototype, **filter it locally first**. Keep the top 20,000 movies based on popularity/vote_count. Replit's RAM will crash if you try to load and embed 1 million movies in memory.

3. **API Keys:** Go to Replit "Secrets" (Lock icon) and add:

### **Phase 1: Data Setup & Cleaning**

**Goal:** Create a clean "soup" of text (Overview + Keywords + Genres) for each movie.

**Prompt for Replit Agent:**

> "I have a file named `movies.csv` uploaded. Write a Python script using pandas to load this data.
> 1. Clean the data by filling missing values in 'overview', 'genres', and 'keywords' with empty strings.
> 2. Create a new column called 'combined_features' that combines the text from 'title', 'genres', 'keywords', and 'overview' into a single string for each movie.
> 3. Display the first 5 rows to verify."
> 
> 

---

### **Phase 2: The Matcher (Sentence-BERT)**

**Goal:** Convert text into numbers (vectors) so we can find similar movies.

**Prompt for Replit Agent:**

> "Now, let's build the embedding engine.
> 1. Install `sentence-transformers` and `scikit-learn`.
> 2. Load the 'all-MiniLM-L6-v2' model.
> 3. Write a function that takes the 'combined_features' column and generates embeddings.
> 4. Compute a Cosine Similarity matrix for the dataset.
> *Note: Since this is a demo, just process the first 5000 movies if memory is tight, or use a memory-efficient approach.*"
> 
> 

**Constraint Note:** Replit might struggle here. If it fails, ask the Agent: *"Use pickle to save the similarity matrix to a file so we don't have to recalculate it every time."*

---

### **Phase 3: The Recommendation Logic**

**Goal:** Input a movie name, output the top 5 matches.

**Prompt for Replit Agent:**

> "Create a recommendation function `recommend(movie_title)`.
> 1. It should find the index of the movie in the dataframe.
> 2. Look up the similarity scores for that movie in our matrix.
> 3. Sort the scores to find the top 5 most similar movies (excluding the movie itself).
> 4. Return the titles and overviews of these 5 movies."
> 
> 

---

### **Phase 4: The Explanation Engine (Gemini API)**

**Goal:** The "Why". This is the unique part of your project.

**Prompt for Replit Agent:**

> "I need an Explanation Engine using the Google Gemini API.
> 1. Install `google-generativeai`.
> 2. Configure it using the `GEMINI_API_KEY` from Replit Secrets.
> 3. Create a function `explain_recommendation(source_movie, recommended_movie, source_plot, recommended_plot)`.
> 4. Inside, send a prompt to Gemini: 'I recommended [recommended_movie] to a user who likes [source_movie]. Here are their plots: [source_plot] and [recommended_plot]. Explain why they are similar in one short, exciting sentence.'
> 5. Return the text generated by Gemini."
> 
>

### **Why this specific plan?**

1. **Memory Management:** Replit has limited RAM. By advising you to filter the dataset first (Phase 1) and use pickle (Phase 2), we avoid "Out of Memory" crashes.
2. **Hybrid Approach:** We use **SBERT for speed** (finding the movie) and **Gemini for intelligence** (explaining the movie). If you used Gemini to *find* the movies, it would be too slow and expensive.
3. **Modular Building:** By asking the Agent to build function-by-function, debugging is easier than asking for one giant block of code.
